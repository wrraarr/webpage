How It Works

  1. Enter your CLOs Open the Data Entry tool. Add your course number, title, and a short description. Then add each Course Learning Outcome — the specific, measurable things students will be able to do after completing your course.
  2. Link CLOs to PLOs For each CLO, use the 8 color-coded category buttons to browse PLOs. Click chips to select alignments. Hover any chip to read the full PLO text. Most CLOs map to 1–4 PLOs.
  3. Review and send Click "Send to Mapper" to preview your data. The preview shows warnings for incomplete entries. Confirm, and the mapper opens with your course loaded automatically.
  4. Explore the map Your courses appear connected to PLOs through their CLOs. Click nodes to expand, drag to rearrange, right-click to pin. Open the PLO Coverage histogram to see which outcomes are well-covered and which have gaps.

Program Learning Outcomes — 8 Categories

The Informatics BS Program Learning Outcomes are organized into 8 competency categories with 43 specific, assessable sub-outcomes.

Core Knowledge

Info systems, data structures, HCI, sociotechnical analysis, ethics, policy. 6 PLOs.

Technical Practice

Programming, frameworks, version control, prototyping, databases, cloud, accessibility. 7 PLOs.

Data & Analysis

Statistical methods, qualitative research, data quality, visualization, analytical tools. 5 PLOs.

Problem Solving

Human-centered design, feasibility, stakeholder balance, iteration, scale. 5 PLOs.

Communication

Written docs, oral presentations, visual communication, diverse audiences. 4 PLOs.

Collaboration

Project management, roles, integrating expertise, feedback, conflict resolution. 5 PLOs.

Ethics & Responsibility

Privacy, algorithmic bias, accessibility, sustainability, equity, professional codes. 6 PLOs.

Lifelong Learning

Independent learning, critical evaluation of tech, professional community, reflection. 5 PLOs.

JSON Format Reference

PLO IDs follow category-index — for example knowledge-0, ethics-3, technical-6. Indexes start at 0 within each category. Use the GPT Prompt Guide to generate this JSON automatically from a syllabus.

{ "courses": [ { "courseNumber": "INFO-I 101", "courseTitle": "Introduction to Informatics", "description": "A ~50-word course description...", "clos": [ { "text": "Decompose real-world problems into sub-problems", "ploMappings": ["problem-0", "knowledge-1"] } ] } ] }

Tips for Good Mappings

  1. Start with action verbs Write CLOs that begin with measurable verbs: analyze, construct, evaluate, design, implement. This makes it easier to identify which PLOs align.
  2. Aim for 1–4 PLO mappings per CLO If a CLO maps to more than 4 PLOs it's probably too broad — consider splitting it. If it maps to zero, it may not align with program goals.
  3. Check cross-category coverage Many courses naturally span multiple categories. A technical course might also touch Communication or Ethics — don't overlook those connections.
  4. Use the coverage histogram After loading your courses, open the PLO Coverage panel. Look for PLOs with zero hits (gaps) and PLOs with unusually high counts (potential over-emphasis).
